# VoiceBar Walkthrough Recording Checklist

## Preflight

- Confirm the recording uses a sanitized demo account or generated visuals.
- Close messaging, e-mail, calendars, private browser tabs, and private repository pages.
- Disable notifications.
- Use synthetic snippets only.
- Use synthetic document text only.
- Do not show private paths, local logs, transcripts, screenshots, recordings, or real command bodies.
- Confirm the README file and video script agree on Mac-first and Windows-roadmap wording.

## Demo Beats

- Show VoiceBar as a local-first Mac voice workflow tool.
- Show Function key (Fn) push-to-speak only if it is working on the recording machine.
- If Function key (Fn) behavior is not reliable, show fallback shortcut usage instead and say so plainly.
- Show dictation insertion with synthetic text.
- Show snippet expansion with synthetic text.
- Show command-text safety as insertion only, without execution.
- Show Read Selection or Read Clipboard with synthetic text.
- Show Apache License 2.0 direction and third-party notices requirement.

## Claims To Avoid

- Do not claim Windows support works.
- Do not claim iPhone support works.
- Do not claim the repository is already public.
- Do not claim legal readiness.
- Do not claim certification, compliance, or enterprise readiness.
- Do not claim hosted publication or release package availability unless verified.

## Post-Recording Review

- Watch the full recording once for private information.
- Check every visible uniform resource locator (URL), file path, window title, and snippet value.
- Confirm all examples are synthetic.
- Confirm no private repository pull request (PR), issue, review thread, or dashboard is visible.
- Confirm no credentials, tokens, e-mail addresses, contact details, or personal data are visible.
- Store raw recording assets outside the repository unless a future lane intentionally adds sanitized generated assets.

## Remotion Follow-Up

- Keep any Remotion project isolated under a dedicated path.
- Add generated video outputs to ignore rules before rendering.
- Run a still-frame check or preview validation before calling a render usable.
- Record validation commands and output in the future Remotion production prompt artifact.
