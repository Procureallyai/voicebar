## Voiceover Script

Voice workflows are fast, but they often ask you to trust opaque cloud systems with sensitive text.

VoiceBar takes a different approach. It is a local-first Mac voice workflow tool for privacy-conscious Artificial Intelligence (AI) productivity.

It was built by Live, an Artificial Intelligence (AI) engineer and machine learning engineer working across governance, policy, guardrails, and practical automation.

The idea is simple: productivity automation should stay human-controlled. Dictation should be local where practical. Snippets should expand text, not silently execute commands. And Artificial Intelligence (AI) workflows should include evidence, governance, and user agency from day one.

Here is the everyday flow.

Hold the Function key (Fn), or a fallback shortcut if your keyboard or macOS setup needs one. Speak a short note. Release. VoiceBar transcribes locally through `whisper.cpp`, then inserts the text.

For snippets, use explicit trigger phrases and aliases. This synthetic example expands a short project update. Even command-text snippets insert text only; VoiceBar does not press Return or execute shell commands automatically.

For reading, select text or use the clipboard, then have VoiceBar read it back through the local text-to-speech path.

VoiceBar is also connected to a wider evidence-first philosophy. Digital Dance AI (DDAI) helps organisations deploy practical Artificial Intelligence (AI) workflows with governance built in. Evidary is being built for Artificial Intelligence (AI) governance, policy, and audit-readiness. VoiceBar is a public proof-of-work for those values in a daily tool.

VoiceBar is being prepared for release under Apache License 2.0 with maintainer-led pull request (PR) review and third-party notices before public release.
